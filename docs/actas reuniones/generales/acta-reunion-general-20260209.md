## Desarrollo de la reunión

Guillermo Ciria inicia la reunión a las 20:03, indicando que no hay un itinerario fijo, sino que se informará sobre el estado de los casos de uso y próximas actividades. Se comenta que algunos casos de uso no son core, redactados por el grupo de trabajo de la idea principal y se decide compartir el documento de Google para facilitar la revisión.

Marta explica las nuevas funcionalidades añadidas, especialmente la diferenciación entre arrendador y arrendatario, y el proceso de selección de objetos por parte del arrendatario. Se detalla el sistema de precios, donde cada objeto tiene un rango definido y el arrendador elige el precio dentro de ese rango. Además, se describe el sistema de envío, que puede ser por mensajería o punto de encuentro, similar a Wallapop.

Se aclara que los objetos de un kit pueden llegar en diferentes días y por distintos sistemas de envío, y que la aplicación está pensada para alquileres planificados, no envíos rápidos. La idea es operar a nivel europeo, aunque se prioriza la logística local para evitar complicaciones y altos costes.

Se introduce el sistema de fianza, que retiene una cantidad a ambas partes hasta que se verifica la correcta entrega y devolución del objeto. Paula y Marta proponen definir los casos de uso core para avanzar con los mockups.

Lucía informa sobre el avance en la investigación del stack tecnológico, mencionando Capacitor para el frontend y discutiendo opciones como Spring Boot, Django, y NestJS para el backend. Se debate sobre el uso de Kotlin y las ventajas de Spring Boot. Se decide realizar una votación por Whatsapp para elegir el stack tecnológico, resultando ganadores Java para backend y React Native con TypeScript para frontend. Para la base de datos se opta por PostgreSQL.

Se acuerda dockerizar la aplicación para facilitar el despliegue, y se discuten alternativas como Vagrant. Se destaca la importancia de estudiar las plataformas de despliegue y los servicios disponibles en las ciudades objetivo.

Sobre la organización, se decide priorizar los mockups y casos de uso core para la semana, dejando la formación definitiva de equipos y la organización para la siguiente semana. Se plantea la posibilidad de permitir alquileres individuales, aunque se advierte sobre la complejidad logística.

Se concluye la reunión a las 20:58, proponiendo la siguiente reunión para el lunes siguiente a las 20:00.
